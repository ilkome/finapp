Future optimization plans for the loans feature.

## markOverduePayments Cron Optimization

**Problem:** The daily cron `markOverduePayments` in `convex/loans.ts` scans ALL loans across ALL users with `ctx.db.query('loans').collect()`, then for each active loan queries all its payments. This is O(L*P) where L = total loans, P = average payments per loan.

**Current state:** Works fine for small user counts (< 100 users). No functional issue.

**When to optimize:** When the total number of active loans exceeds ~1000.

**Proposed solution:**

1. Add a global `by_status_date` index on `loanPayments` table: `['status', 'date']`
2. Replace the double-loop with a single query:

```ts
ctx.db
  .query('loanPayments')
  .withIndex('by_status_date', q =>
    q.eq('status', 'scheduled').lt('date', Date.now()),
  )
  .collect()
```

3. Batch-patch all matching payments to `'overdue'`
4. This reduces complexity from O(L*P) to O(overdue) - only touching payments that actually need updating

**Schema change required:**

```ts
// In convex/schema.ts, add to loanPayments:
.index('by_status_date', ['status', 'date'])
```
