> **Historical document.** This page describes a planned optimization for `convex/loans.ts`, a file that existed in the Convex era of Finapp. The Convex backend has been replaced by Supabase Postgres + PowerSync. There is no loans table or feature in the current app - `convex/loans.ts` does not exist. The algorithmic discussion below is kept as a historical reference only.

## markOverduePayments Cron Optimization (Convex era)

**Problem:** The daily cron `markOverduePayments` in `convex/loans.ts` scanned ALL loans across ALL users with `ctx.db.query('loans').collect()`, then for each active loan queried all its payments. This is O(L*P) where L = total loans, P = average payments per loan.

**When to optimize:** When the total number of active loans exceeds ~1000.

**Proposed solution:**

1. Add a global `by_status_date` index on `loanPayments` table: `['status', 'date']`
2. Replace the double-loop with a single query:

```ts
ctx.db
  .query('loanPayments')
  .withIndex('by_status_date', q =>
    q.eq('status', 'scheduled').lt('date', Date.now()),
  )
  .collect()
```

3. Batch-patch all matching payments to `'overdue'`
4. This reduces complexity from O(L*P) to O(overdue) - only touching payments that actually need updating

**Schema change required (Convex era, no longer applicable):**

```ts
// In convex/schema.ts, add to loanPayments:
.index('by_status_date', ['status', 'date'])
```
