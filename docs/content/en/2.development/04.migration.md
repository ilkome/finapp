Finapp has gone through two major backend changes. This page documents both for historical reference.

## Firebase to Convex (v6 to v7)

Finapp v6 used Firebase Realtime Database. Finapp v7 replaced it with Convex (backend-as-code, WebSocket sync).

The legacy v6 source code lives on the [`firebase`](https://github.com/ilkome/finapp/tree/firebase) branch.

### What changed

| Entity | Firebase (v6) | Convex (v7) |
|--------|---------------|-------------|
| Accounts | `accounts` with string keys | `wallets` table with Convex IDs |
| Categories | String IDs, `parentId` as string | Convex IDs, `parentId` as `Id<'categories'>` or `0` |
| Transactions | String IDs, references by old keys | Convex IDs, all references remapped |
| Settings | `settings.lang` (`en`, `ru`) | `userSettings.locale` (`en`, `ru`) |
| Icons | `mdi mdi-xxx` format | `mdi:xxx` format |

### Migration tooling (historical - no longer in the repo)

This migration was a one-off operation. The scripts that performed it (`utils/migrate/firebase.js`, `convex/migrate.ts`) have been deleted from the repository because they are no longer relevant. Do not attempt to run them.

The scripts read a Firebase JSON export, transformed the data to match the Convex schema (remapping IDs, fixing icon formats, normalizing settings), and imported it via Convex internal mutations in batches of 200.

## Convex to Supabase+PowerSync (current backend)

Finapp v7+ (Convex) has been migrated to **Supabase Postgres + PowerSync** (offline-first sync) with **Supabase Auth (email/password)**.

This was a one-off development-time operation, not a recurring user-facing flow:

1. Export all data from the Convex deployment using `convex export --prod`.
2. Transform the export with a Python script that kept Convex `_id` values as Postgres `text` ids (no UUID conversion needed - the ids remain stable) and remapped `userId` values to the corresponding Supabase Auth uids.
3. Import via `psql` into the Supabase Postgres instance.

No FK constraints exist in the Postgres schema, so import order does not matter. RLS policies and PowerSync sync rules ensure per-user data isolation going forward.

## Current backend

The current backend is **Supabase Postgres + PowerSync**. See [Architecture](/en/reference/architecture) for a full description and [Installation](/en/development/installation) for local setup.
