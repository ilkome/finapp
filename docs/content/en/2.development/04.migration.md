Finapp's backend was migrated from **Firebase Realtime Database** to **Supabase Postgres + PowerSync** (offline-first sync) with **Supabase Auth (email/password)**. This page documents that one-off migration for historical reference.

The legacy Firebase (v6) source code lives on the [`firebase`](https://github.com/ilkome/finapp/tree/firebase) branch.

## What changed

| Entity | Firebase (v6) | Current (Supabase + PowerSync) |
|--------|---------------|--------------------------------|
| Accounts | `accounts` with string keys | `wallets` table, `text` ids |
| Categories | String IDs, `parentId` as string | `text` ids, `parentId` as a category id or `0` |
| Transactions | String IDs, references by old keys | `text` ids, all references remapped |
| Settings | `settings.lang` (`en`, `ru`) | `user_settings.locale` (`en`, `ru`) |
| Icons | `mdi mdi-xxx` format | `mdi:xxx` format |

## Migration tooling (historical - no longer in the repo)

This was a one-off development-time operation, not a recurring user-facing flow. The scripts that performed it have been deleted from the repository because they are no longer relevant. Do not attempt to run them.

The process:

1. Export the data from Firebase as JSON.
2. Transform the export with a script that remapped ids to stable `text` ids, fixed icon formats, normalized settings, and mapped users to their Supabase Auth uids.
3. Import via `psql` into the Supabase Postgres instance.

No FK constraints exist in the Postgres schema, so import order does not matter. RLS policies and PowerSync sync rules ensure per-user data isolation going forward.

## Current backend

The current backend is **Supabase Postgres + PowerSync**. See [Architecture](/en/reference/architecture) for a full description and [Installation](/en/development/installation) for local setup.
