## Requirements

- [Node.js](https://nodejs.org) v24.12.0 or later
- [pnpm](https://pnpm.io) >= 10.28.2

## Installation

::steps
### Clone the repository

```bash [Terminal]
git clone https://github.com/ilkome/finapp.git
cd finapp
```

### Install dependencies

```bash [Terminal]
pnpm install
```

### Start development server

You need two terminals — one for the frontend and one for the backend.

```bash [Terminal 1 — Frontend]
pnpm dev
```

```bash [Terminal 2 — Backend]
pnpm dev:convex
```

The app will be available at [http://localhost:3050](http://localhost:3050).

A design system playground is available at [http://localhost:3050/dev](http://localhost:3050/dev) during development (hidden in production).
::

## Env Files

- `.env` — dev deployment values, read by Nuxt (default `--dotenv`)
- `.env.local` — auto-created by `convex dev`, used by Convex CLI (gitignored)
- `.env.prod` — prod deployment values, used by `pnpm dev:prod`

All `.env*` files are gitignored except `.env.example`.

## Backend Setup

Finapp uses [Convex](https://convex.dev) as a backend. To set up your own instance:

1. Create a project at [convex.dev](https://convex.dev)
2. Copy `.env.example` to `.env` and fill in your Convex credentials:

```bash [.env]
CONVEX_DEPLOYMENT=your_deployment
VITE_CONVEX_URL=your_convex_url
VITE_CONVEX_SITE_URL=your_convex_site_url
```

### Nuxt Env Vars

| Variable | Description |
|----------|-------------|
| `CONVEX_DEPLOYMENT` | Deployment identifier for Convex CLI |
| `VITE_CONVEX_URL` | Convex deployment URL (Nuxt frontend) |
| `VITE_CONVEX_SITE_URL` | Convex site URL (Nuxt auth client) |

### Convex Backend Env Vars

Set via `npx convex env set`:

| Variable | Description |
|----------|-------------|
| `BETTER_AUTH_SECRET` | Secret key for Better Auth (required — without it all auth endpoints including CORS preflight will fail) |
| `APP_URL` | App URL for cross-domain auth and CORS/trustedOrigins |
| `GOOGLE_CLIENT_ID` | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret |
| `OPENEXCHANGERATES_APP_ID` | Exchange rates API key |

### First-Time Convex Setup

When connecting to a new Convex dev deployment, configure its environment:

```bash [Terminal]
npx convex env set BETTER_AUTH_SECRET <value>
npx convex env set APP_URL http://localhost:3050
npx convex env set GOOGLE_CLIENT_ID <value>
npx convex env set GOOGLE_CLIENT_SECRET <value>
npx convex env set OPENEXCHANGERATES_APP_ID <value>
```

::callout{icon="i-lucide-alert-triangle" color="warning"}
`BETTER_AUTH_SECRET` is required — without it, all auth endpoints (including CORS preflight) will fail with `BetterAuthError`.
::
