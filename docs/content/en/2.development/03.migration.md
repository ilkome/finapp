Finapp v6 used Firebase Realtime Database. Finapp v7 uses [Convex](https://convex.dev). This guide describes how to migrate existing user data.

## Overview

The migration script reads a Firebase JSON export, transforms the data to match the Convex schema, and imports it via internal mutations.

### What changes

| Entity | Firebase (v6) | Convex (v7) |
|--------|---------------|-------------|
| Accounts | `accounts` with string keys | `wallets` table with Convex IDs |
| Categories | String IDs, `parentId` as string | Convex IDs, `parentId` as `Id<'categories'>` or `0` |
| Transactions | String IDs, references by old keys | Convex IDs, all references remapped |
| Settings | `settings.lang` (`en`, `ru`) | `userSettings.locale` (`en`, `ru`) |
| Icons | `mdi mdi-xxx` format | `mdi:xxx` format |

## Prerequisites

- Convex backend deployed (both dev and prod)
- User account created on the target deployment (sign in at least once to get a userId)
- Firebase JSON export file

## Export data from Firebase

Export your data from the [Firebase Console](https://console.firebase.google.com):

1. Open your Firebase project
2. Go to Realtime Database
3. Click the three-dot menu on the root node for the user
4. Select "Export JSON"
5. Save the file as `data.json` in `utils/migrate/`

The file should have this structure:

```json [data.json]
{
  "accounts": { ... },
  "categories": { ... },
  "settings": { "baseCurrency": "USD", "lang": "en" },
  "trns": { ... }
}
```

## Find your Convex userId

You need the Convex userId for the target user on each deployment. Find it in the [Convex Dashboard](https://dashboard.convex.dev) in the `users` table.

## Run the migration

The script deletes existing user data before importing.

::callout{icon="i-lucide-alert-triangle" color="warning"}
The script will delete all existing data for the specified userId before importing. Make sure you have the correct userId.
::

```bash [Dev]
node utils/migrate/firebase.js <userId>
```

```bash [Prod]
node utils/migrate/firebase.js <userId> --prod
```

The script will:

1. Delete existing data for the userId
2. Import wallets (from `accounts`) and return an ID mapping
3. Import categories with icon fixes and parent/child remapping
4. Import transactions in batches of 200, remapping all wallet and category references
5. Import user settings

## Data transformations

The script applies these transformations during import:

### Wallets
- `accounts` key renamed to `wallets`
- Missing fields get defaults: `isArchived: false`, `isExcludeInTotal: false`, `isWithdrawal: false`, `order: 0`

### Categories
- Icons: `mdi mdi-xxx` → `mdi:xxx`, empty → `mdi:category-outline`
- `parentId` remapped from old string IDs to new Convex IDs (`childIds` no longer stored — computed from `parentId`)
- Missing fields get defaults: `showInLastUsed: false`, `showInQuickSelector: false`

### Transactions
- `walletId`, `categoryId`, `expenseWalletId`, `incomeWalletId` remapped to new Convex IDs
- Special `categoryId` values preserved as-is: `transfer`, `adjustment`

### Settings
- `settings.lang` → `userSettings.locale` (values stay the same: `en`, `ru`)

## Files

| File | Description |
|------|-------------|
| `utils/migrate/firebase.js` | Node.js script that reads JSON, transforms data, calls Convex mutations |
| `convex/migrate.ts` | Convex internal mutations: `deleteUserData`, `insertWallets`, `insertCategories`, `insertTrnsBatch`, `insertUserSettings` |
