Finapp is a pnpm workspace monorepo. Both the app and docs deploy from the same repository as separate Vercel projects.

## Vercel Setup

Create **two projects** in Vercel, both pointing to the same GitHub repository.

### App Project

| Setting | Value |
|---------|-------|
| Root Directory | `app` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |
| Install Command | `pnpm install` |

Add the following environment variables:

| Variable | Value |
|----------|-------|
| `VITE_CONVEX_URL` | Your Convex deployment URL |
| `VITE_CONVEX_SITE_URL` | Your Convex site URL |

### Docs Project

| Setting | Value |
|---------|-------|
| Root Directory | `docs` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |

## Ignored Build Step

To avoid unnecessary builds, configure **Ignored Build Step** in each Vercel project settings (Settings → Git → Ignored Build Step):

::code-group
```bash [App]
git diff --quiet HEAD^ HEAD -- . ':!docs'
```

```bash [Docs]
git diff --quiet HEAD^ HEAD -- docs
```
::

This way the app won't redeploy when only docs change, and vice versa.

## Premium (Finapp Premium)

Finapp Premium lives in a separate repository ([finapp-premium](https://github.com/ilkome/finapp-premium)) and extends the base app as a **Nuxt layer**.

### Vercel Settings

| Setting | Value |
|---------|-------|
| Root Directory | `.` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm generate` |
| Output Directory | `.output/public` |
| Install Command | `pnpm install` |

No `FINAPP_BASE_PATH` environment variable is needed on Vercel. The default path is `./base/app`, which points to the checked-out base submodule.

### How It Works

- `nuxt.config.ts` uses `FINAPP_BASE_PATH` env var to resolve the base layer
- Default (CI/Vercel): `./base/app` — the `app/` subdirectory inside the `base` Git submodule
- `base/` is a Git submodule pointing to the `convex` branch of `ilkome/finapp`
- Local dev: set `FINAPP_BASE_PATH=../mono/app` in `.env` to use the local monorepo with hot reload

::callout{icon="i-lucide-info" color="info"}
Vercel initializes Git submodules during checkout. The base submodule repository must be public.
::

### Local Development

Set `FINAPP_BASE_PATH` in `.env` to point to the monorepo app directory:

```bash [.env]
FINAPP_BASE_PATH=../mono/app
```

### Git Submodule

The submodule should point to the `convex` branch of `ilkome/finapp`:

```bash [Terminal]
git submodule add -b convex https://github.com/ilkome/finapp.git base
```

After a fresh clone:

```bash [Terminal]
git submodule update --init
```

## Convex Backend

Convex deploys separately from Vercel. Use the Convex dashboard or CLI:

```bash [Terminal]
npx convex deploy
```

::callout{icon="i-lucide-info" color="info"}
Make sure all required environment variables are set in your Convex production deployment. See [Installation](/development/installation#convex-backend-env-vars) for the full list.
::
