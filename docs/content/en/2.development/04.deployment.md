Finapp is a pnpm workspace monorepo. Both the app and docs deploy from the same repository as separate Vercel projects.

## Vercel Setup

Create **two projects** in Vercel, both pointing to the same GitHub repository.

### App Project

| Setting | Value |
|---------|-------|
| Root Directory | `.` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |

Add the following environment variables:

| Variable | Value |
|----------|-------|
| `VITE_CONVEX_URL` | Your Convex deployment URL |
| `VITE_CONVEX_SITE_URL` | Your Convex site URL |

### Docs Project

| Setting | Value |
|---------|-------|
| Root Directory | `docs` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |

## Ignored Build Step

To avoid unnecessary builds, configure **Ignored Build Step** in each Vercel project settings (Settings → Git → Ignored Build Step):

::code-group
```bash [App]
git diff --quiet HEAD^ HEAD -- . ':!docs'
```

```bash [Docs]
git diff --quiet HEAD^ HEAD -- docs
```
::

This way the app won't redeploy when only docs change, and vice versa.

## Premium (Finapp Premium)

Finapp Premium lives in a separate repository ([finapp-premium](https://github.com/ilkome/finapp-premium)) and uses the base app as a **Git submodule** + Nuxt layer.

### Vercel Settings

| Setting | Value |
|---------|-------|
| Root Directory | `.` |
| Framework Preset | Nuxt.js |
| Build Command | `npx nuxt prepare base && pnpm generate` |
| Output Directory | `.output/public` |
| Install Command | `pnpm install` |

::callout{icon="i-lucide-info" color="info"}
Vercel automatically initializes Git submodules during checkout. The base submodule repository must be public.
::

### How It Works

- `base/` is a Git submodule pointing to the `convex` branch of `ilkome/finapp`
- `nuxt.config.ts` uses `FINAPP_BASE_PATH` env var (default: `./base`) to resolve the base layer
- `npx nuxt prepare base` generates `base/.nuxt/tsconfig.json` — required for Vite to resolve types from the submodule
- The `pnpm dev` script auto-runs `nuxt prepare base` if `base/.nuxt` doesn't exist

### Local Development

For local development, set `FINAPP_BASE_PATH=../base` in `.env` to use the sibling directory with live hot reload instead of the submodule.

```bash [.env]
FINAPP_BASE_PATH=../base
```

To test with the submodule locally:

```bash [Terminal]
FINAPP_BASE_PATH=./base pnpm dev
```

### Updating the Submodule

```bash [Terminal]
git submodule update --remote base
```

After a fresh clone:

```bash [Terminal]
git submodule update --init
```

## Convex Backend

Convex deploys separately from Vercel. Use the Convex dashboard or CLI:

```bash [Terminal]
npx convex deploy
```

::callout{icon="i-lucide-info" color="info"}
Make sure all required environment variables are set in your Convex production deployment. See [Installation](/development/installation#convex-backend-env-vars) for the full list.
::
