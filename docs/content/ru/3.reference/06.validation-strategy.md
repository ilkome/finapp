## Текущий подход: двойная валидация

В проекте две независимые системы валидации:

| Слой | Фреймворк | Назначение |
|------|-----------|------------|
| **Клиент** (браузер) | Zod v4 | UX форм - дефолты, трим, мгновенные ошибки |
| **Сервер** (Convex) | `convex/values` (`v.*`) | Целостность данных - типизированные ID-ссылки, проверка владельца |

### Где живёт валидация

| Сущность | Convex-валидаторы | Zod-схемы |
|----------|-------------------|-----------|
| Кошельки | `convex/wallets.ts`, `convex/schema.ts` | `app/components/wallets/types.ts` |
| Категории | `convex/categories.ts`, `convex/schema.ts` | `app/components/categories/types.ts` |
| Транзакции | `convex/trns.ts`, `convex/schema.ts` | `app/components/trns/types.ts` |
| Настройки | `convex/userSettings.ts` | - (нет клиентской схемы) |
| Курсы валют | `convex/rates.ts` | - (нет клиентской схемы) |
| Конфиг статистики | - | `app/components/stat/useStatConfig.ts` |
| Параметры дат | - | `app/components/date/useStatDate.ts` |

### Что валидирует каждая сторона

**Zod (только клиент):**
- `.trim().min(1)` на имени/иконке - не пропускает пустые после трима
- `.positive()` на суммах
- `.default()` для начальных значений форм
- `.transform()` для парсинга query-параметров (строка → число)

**Convex (только сервер):**
- `v.id('wallets')`, `v.id('categories')` - типизированные ссылки на записи
- Проверка владельца сущности (`userId`)
- Защита от циклических ссылок в категориях
- Уникальность имени категории в рамках родителя
- Ограничение вложенности категорий (2 уровня)
- Проверка что кошельки трансфера различаются
- Лимиты длины строк: name (100), description (500), icon (50), color (50), currency (10)
- Числовые диапазоны: amount (макс. 1 трлн), creditLimit (макс. 1 млрд), date (2000–2099)
- Контроль `categoryId`: для Transfer обязателен `'transfer'`, для не-Transfer запрещён `'transfer'`

**Обе стороны (дублирование ~40%):**
- Наличие / опциональность полей
- Базовые проверки типов (string, number, boolean)
- Валидация union / literal (TrnType, WalletType)

### Почему константы дублируются

Enum `TrnType` и массив `walletTypes` определены независимо на каждой стороне:
- `convex/validators.ts` - для сервера (импортирует `v` из `convex/values`)
- `app/components/trns/types.ts` / `app/components/wallets/types.ts` - для клиента

Импорт из директории `convex/` в браузерный код тянет серверные модули (`_generated/server`, auth) и вызывает предупреждение «Convex functions should not be imported in the browser». Подробнее в [технических решениях](/ru/development/tech-decisions).

## Рассмотренные альтернативы

### fluent-convex + общие Zod-схемы

[fluent-convex](https://github.com/mikecann/fluent-convex) (v0.x) - fluent API для Convex-функций с Zod-плагином, который автоматически конвертирует Zod-схемы в Convex-валидаторы.

**Что бы дало:** одна Zod-схема, используемая и на клиенте, и на сервере.

**Отклонено, потому что:**
- Библиотека экспериментальная (v0.x, один автор, 90 звёзд)
- `v.id('wallets')` не имеет аналога в Zod - теряются типизированные ID-ссылки
- `schema.ts` (определение таблиц) всё равно остаётся на `v.*`
- Дополнительная зависимость: `convex-helpers` как peer dep

### Общие Zod-схемы с ручной конвертацией через convex-helpers

Определить Zod-схемы в директории `shared/`. Клиент использует напрямую. Сервер конвертирует через `convex-helpers/server/zod`.

**Что бы дало:** единый источник правды без зависимости от fluent-convex.

**Отклонено, потому что:**
- Та же проблема с `v.id()` - нет Zod-эквивалента
- Нужно настраивать tsconfig Convex для импорта из-за пределов `convex/`
- Бойлерплейт для ручной конвертации Zod → Convex
- Минимальная выгода для 3 стабильных сущностей с ~15 полями каждая

## Решение

Оставить двойную валидацию. Дублирование невелико (~15 полей × 3 сущности), схемы меняются редко, и каждая сторона решает свою задачу:
- **Zod** = UX (дефолты, трансформации, мгновенная обратная связь)
- **Convex** = целостность (ID-ссылки, владелец, бизнес-правила)

### Когда пересмотреть

- Поддержка `v.id()` в `convex-helpers/server/zod`
- Выход fluent-convex v1.0 со стабильным API
