Finapp — это монорепа на pnpm workspaces. Приложение и документация деплоятся из одного репозитория как два отдельных проекта в Vercel.

## Настройка Vercel

Создайте **два проекта** в Vercel, оба указывают на один GitHub-репозиторий.

### Проект приложения

| Настройка | Значение |
|-----------|----------|
| Root Directory | `app` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |
| Install Command | `pnpm install` |

Добавьте переменные окружения:

| Переменная | Значение |
|------------|----------|
| `VITE_CONVEX_URL` | URL вашего Convex-деплоймента |
| `VITE_CONVEX_SITE_URL` | Site URL вашего Convex-деплоймента |

### Проект документации

| Настройка | Значение |
|-----------|----------|
| Root Directory | `docs` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |

## Ignored Build Step

Чтобы избежать лишних билдов, настройте **Ignored Build Step** в каждом проекте Vercel (Settings → Git → Ignored Build Step):

::code-group
```bash [App]
git diff --quiet HEAD^ HEAD -- . ':!docs'
```

```bash [Docs]
git diff --quiet HEAD^ HEAD -- docs
```
::

Так приложение не будет передеплоиваться при изменениях только в документации, и наоборот.

## Premium (Finapp Premium)

Finapp Premium живёт в отдельном репозитории ([finapp-premium](https://github.com/ilkome/finapp-premium)) и расширяет базовое приложение как **Nuxt layer**.

### Настройки Vercel

| Настройка | Значение |
|-----------|----------|
| Root Directory | `.` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm generate` |
| Output Directory | `.output/public` |
| Install Command | `pnpm install` |

На Vercel не нужно задавать `FINAPP_BASE_PATH`. Путь по умолчанию — `./base/app`, то есть checked-out базовый субмодуль.

### Как это работает

- `nuxt.config.ts` использует переменную `FINAPP_BASE_PATH` для определения пути к базовому слою
- По умолчанию (CI/Vercel): `./base/app` — поддиректория `app/` внутри Git-субмодуля `base`
- `base/` — Git-субмодуль, указывающий на ветку `convex` репозитория `ilkome/finapp`
- Локально: установите `FINAPP_BASE_PATH=../mono/app` в `.env` для работы с локальной монорепой и hot reload

::callout{icon="i-lucide-info" color="info"}
Vercel инициализирует Git-субмодули при checkout. Репозиторий базового субмодуля должен быть публичным.
::

### Локальная разработка

Установите `FINAPP_BASE_PATH` в `.env`, указав на директорию приложения в монорепе:

```bash [.env]
FINAPP_BASE_PATH=../mono/app
```

### Git-субмодуль

Субмодуль должен указывать на ветку `convex` репозитория `ilkome/finapp`:

```bash [Terminal]
git submodule add -b convex https://github.com/ilkome/finapp.git base
```

После свежего клонирования:

```bash [Terminal]
git submodule update --init
```

## Бэкенд Convex

Convex деплоится отдельно от Vercel. Используйте дашборд Convex или CLI:

```bash [Terminal]
npx convex deploy
```

::callout{icon="i-lucide-info" color="info"}
Убедитесь, что все необходимые переменные окружения установлены в продакшен-деплойменте Convex. Полный список — в разделе [Установка](/development/installation#convex-backend-env-vars).
::
