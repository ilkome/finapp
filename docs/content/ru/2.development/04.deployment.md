Finapp — это монорепа на pnpm workspaces. Приложение и документация деплоятся из одного репозитория как два отдельных проекта в Vercel.

## Настройка Vercel

Создайте **два проекта** в Vercel, оба указывают на один GitHub-репозиторий.

### Проект приложения

| Настройка | Значение |
|-----------|----------|
| Root Directory | `.` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |

Добавьте переменные окружения:

| Переменная | Значение |
|------------|----------|
| `VITE_CONVEX_URL` | URL вашего Convex-деплоймента |
| `VITE_CONVEX_SITE_URL` | Site URL вашего Convex-деплоймента |

### Проект документации

| Настройка | Значение |
|-----------|----------|
| Root Directory | `docs` |
| Framework Preset | Nuxt.js |
| Build Command | `pnpm build` |
| Output Directory | `.output/public` |

## Ignored Build Step

Чтобы избежать лишних билдов, настройте **Ignored Build Step** в каждом проекте Vercel (Settings → Git → Ignored Build Step):

::code-group
```bash [App]
git diff --quiet HEAD^ HEAD -- . ':!docs'
```

```bash [Docs]
git diff --quiet HEAD^ HEAD -- docs
```
::

Так приложение не будет передеплоиваться при изменениях только в документации, и наоборот.

## Premium (Finapp Premium)

Finapp Premium живёт в отдельном репозитории ([finapp-premium](https://github.com/ilkome/finapp-premium)) и использует базовое приложение как **Git-субмодуль** + Nuxt layer.

### Настройки Vercel

| Настройка | Значение |
|-----------|----------|
| Root Directory | `.` |
| Framework Preset | Nuxt.js |
| Build Command | `npx nuxt prepare base && pnpm generate` |
| Output Directory | `.output/public` |
| Install Command | `pnpm install` |

::callout{icon="i-lucide-info" color="info"}
Vercel автоматически инициализирует Git-субмодули при клонировании. Репозиторий базового субмодуля должен быть публичным.
::

### Как это работает

- `base/` — Git-субмодуль, указывающий на ветку `convex` репозитория `ilkome/finapp`
- `nuxt.config.ts` использует переменную `FINAPP_BASE_PATH` (по умолчанию: `./base`) для определения пути к базовому слою
- `npx nuxt prepare base` генерирует `base/.nuxt/tsconfig.json` — нужен Vite для резолва типов из субмодуля
- Скрипт `pnpm dev` автоматически запускает `nuxt prepare base`, если `base/.nuxt` не существует

### Локальная разработка

Для локальной разработки установите `FINAPP_BASE_PATH=../base` в `.env`, чтобы использовать соседнюю директорию с живым hot reload вместо субмодуля.

```bash [.env]
FINAPP_BASE_PATH=../base
```

Для тестирования с субмодулем локально:

```bash [Terminal]
FINAPP_BASE_PATH=./base pnpm dev
```

### Обновление субмодуля

```bash [Terminal]
git submodule update --remote base
```

После свежего клонирования:

```bash [Terminal]
git submodule update --init
```

## Бэкенд Convex

Convex деплоится отдельно от Vercel. Используйте дашборд Convex или CLI:

```bash [Terminal]
npx convex deploy
```

::callout{icon="i-lucide-info" color="info"}
Убедитесь, что все необходимые переменные окружения установлены в продакшен-деплойменте Convex. Полный список — в разделе [Установка](/development/installation#convex-backend-env-vars).
::
