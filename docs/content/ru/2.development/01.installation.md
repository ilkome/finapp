## Требования

- [Node.js](https://nodejs.org) v24.12.0+
- [pnpm](https://pnpm.io) >= 10.28.2

## Установка

::steps
### Клонируйте репозиторий

```bash [Terminal]
git clone https://github.com/ilkome/finapp.git
cd finapp
```

### Установите зависимости

```bash [Terminal]
pnpm install
```

### Запустите dev-сервер

Два терминала — фронтенд и бэкенд:

```bash [Терминал 1 — Фронтенд]
pnpm dev
```

```bash [Терминал 2 — Бэкенд]
pnpm dev:convex
```

Приложение доступно на [http://localhost:3050](http://localhost:3050).

Playground дизайн-системы доступен на [http://localhost:3050/dev](http://localhost:3050/dev) в режиме разработки (скрыт в продакшене).
::

## Файлы окружения

- `.env` — значения dev-деплоймента, читается Nuxt (по умолчанию `--dotenv`)
- `.env.local` — создаётся автоматически `convex dev`, используется Convex CLI (в gitignore)
- `.env.prod` — значения prod-деплоймента, используется `pnpm dev:prod`

Все `.env*` файлы в gitignore, кроме `.env.example`.

## Настройка бэкенда

Бэкенд — [Convex](https://convex.dev).

1. Создайте проект на [convex.dev](https://convex.dev)
2. Скопируйте `.env.example` в `.env`:

```bash [.env]
CONVEX_DEPLOYMENT=your_deployment
VITE_CONVEX_URL=your_convex_url
VITE_CONVEX_SITE_URL=your_convex_site_url
```

### Переменные Nuxt

| Переменная | Описание |
|------------|----------|
| `CONVEX_DEPLOYMENT` | Идентификатор деплоймента для Convex CLI |
| `VITE_CONVEX_URL` | URL деплоймента Convex (фронтенд Nuxt) |
| `VITE_CONVEX_SITE_URL` | Site URL Convex (авторизация Nuxt) |

### Переменные Convex backend

Устанавливаются через `npx convex env set`:

| Переменная | Описание |
|------------|----------|
| `BETTER_AUTH_SECRET` | Секретный ключ Better Auth (обязателен — без него все auth-эндпоинты, включая CORS preflight, будут падать) |
| `APP_URL` | URL приложения для кросс-доменной авторизации и CORS/trustedOrigins |
| `GOOGLE_CLIENT_ID` | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret |
| `OPENEXCHANGERATES_APP_ID` | API-ключ курсов валют |

### Первичная настройка Convex

При подключении к новому dev-деплойменту Convex настройте его окружение:

```bash [Terminal]
npx convex env set BETTER_AUTH_SECRET <значение>
npx convex env set APP_URL http://localhost:3050
npx convex env set GOOGLE_CLIENT_ID <значение>
npx convex env set GOOGLE_CLIENT_SECRET <значение>
npx convex env set OPENEXCHANGERATES_APP_ID <значение>
```

::callout{icon="i-lucide-alert-triangle" color="warning"}
`BETTER_AUTH_SECRET` обязателен — без него все auth-эндпоинты (включая CORS preflight) будут падать с ошибкой `BetterAuthError`.
::
