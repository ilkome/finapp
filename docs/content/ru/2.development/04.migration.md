Finapp v6 использовал Firebase Realtime Database. Finapp v7 использует [Convex](https://convex.dev). Это руководство описывает миграцию данных пользователей.

## Обзор

Скрипт миграции читает JSON-экспорт из Firebase, трансформирует данные под схему Convex и импортирует через внутренние мутации.

### Что меняется

| Сущность | Firebase (v6) | Convex (v7) |
|----------|---------------|-------------|
| Счета | `accounts` со строковыми ключами | Таблица `wallets` с Convex ID |
| Категории | Строковые ID, `parentId` строкой | Convex ID, `parentId` как `Id<'categories'>` или `0` |
| Транзакции | Строковые ID, ссылки старыми ключами | Convex ID, все ссылки перемаплены |
| Настройки | `settings.lang` (`en`, `ru`) | `userSettings.locale` (`en`, `ru`) |
| Иконки | Формат `mdi mdi-xxx` | Формат `mdi:xxx` |

## Требования

- Задеплоенный Convex бекенд (dev и prod)
- Созданный аккаунт пользователя на целевом деплойменте (нужно войти хотя бы раз, чтобы получить userId)
- JSON-экспорт из Firebase

## Экспорт данных из Firebase

Экспортируйте данные из [Firebase Console](https://console.firebase.google.com):

1. Откройте проект в Firebase
2. Перейдите в Realtime Database
3. Нажмите меню (три точки) на корневом узле пользователя
4. Выберите "Export JSON"
5. Сохраните файл как `data.json` в `utils/migrate/`

Файл должен иметь такую структуру:

```json [data.json]
{
  "accounts": { ... },
  "categories": { ... },
  "settings": { "baseCurrency": "USD", "lang": "en" },
  "trns": { ... }
}
```

## Найдите Convex userId

Нужен Convex userId пользователя на каждом деплойменте. Найти его можно в [Convex Dashboard](https://dashboard.convex.dev) в таблице `users`.

## Запуск миграции

Скрипт удаляет существующие данные пользователя перед импортом.

::callout{icon="i-lucide-alert-triangle" color="warning"}
Скрипт удалит все существующие данные для указанного userId перед импортом. Убедитесь, что userId правильный.
::

```bash [Dev]
node utils/migrate/firebase.js <userId>
```

```bash [Prod]
node utils/migrate/firebase.js <userId> --prod
```

Скрипт выполнит:

1. Удаление существующих данных для userId
2. Импорт кошельков (из `accounts`) с возвратом маппинга ID
3. Импорт категорий с исправлением иконок и ремаппингом parent/child
4. Импорт транзакций батчами по 200, с ремаппингом всех ссылок на кошельки и категории
5. Импорт настроек пользователя

## Трансформации данных

Скрипт применяет следующие трансформации при импорте:

### Кошельки
- Ключ `accounts` переименован в `wallets`
- Отсутствующие поля получают значения по умолчанию: `isArchived: false`, `isExcludeInTotal: false`, `isWithdrawal: false`, `order: 0`

### Категории
- Иконки: `mdi mdi-xxx` → `mdi:xxx`, пустые → `mdi:category-outline`
- `parentId` перемаплен со старых строковых ID на новые Convex ID (`childIds` больше не хранится — вычисляется из `parentId`)
- Отсутствующие поля получают значения по умолчанию: `showInLastUsed: false`, `showInQuickSelector: false`

### Транзакции
- `walletId`, `categoryId`, `expenseWalletId`, `incomeWalletId` перемаплены на новые Convex ID
- Специальные значения `categoryId` сохранены как есть: `transfer`, `adjustment`

### Настройки
- `settings.lang` → `userSettings.locale` (значения остаются теми же: `en`, `ru`)

## Файлы

| Файл | Описание |
|------|----------|
| `utils/migrate/firebase.js` | Node.js скрипт: читает JSON, трансформирует данные, вызывает мутации Convex |
| `convex/migrate.ts` | Внутренние мутации Convex: `deleteUserData`, `insertWallets`, `insertCategories`, `insertTrnsBatch`, `insertUserSettings` |
