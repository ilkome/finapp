Бэкенд Финапки был мигрирован с **Firebase Realtime Database** на **Supabase Postgres + PowerSync** (offline-first синхронизация) с **Supabase Auth (email/password)**. На этой странице эта разовая миграция описана как исторический контекст.

Исходный код Firebase (v6) хранится на ветке [`firebase`](https://github.com/ilkome/finapp/tree/firebase).

## Что изменилось

| Сущность | Firebase (v6) | Текущее (Supabase + PowerSync) |
|----------|---------------|--------------------------------|
| Счета | `accounts` со строковыми ключами | Таблица `wallets`, `text`-идентификаторы |
| Категории | Строковые ID, `parentId` строкой | `text`-идентификаторы, `parentId` как id категории или `0` |
| Транзакции | Строковые ID, ссылки старыми ключами | `text`-идентификаторы, все ссылки перемаплены |
| Настройки | `settings.lang` (`en`, `ru`) | `user_settings.locale` (`en`, `ru`) |
| Иконки | Формат `mdi mdi-xxx` | Формат `mdi:xxx` |

## Инструменты миграции (исторически - больше нет в репозитории)

Это была разовая операция во время разработки, не повторяющийся пользовательский процесс. Скрипты, которые её выполняли, удалены из репозитория как неактуальные. Запускать их не нужно.

Процесс:

1. Экспорт данных из Firebase в JSON.
2. Трансформация экспорта скриптом: ремаппинг id в стабильные `text`-идентификаторы, исправление формата иконок, нормализация настроек и сопоставление пользователей с их Supabase Auth uid.
3. Импорт через `psql` в экземпляр Supabase Postgres.

В схеме Postgres нет FK-ограничений, поэтому порядок импорта не имеет значения. RLS-политики и правила синхронизации PowerSync обеспечивают изоляцию данных по пользователям.
